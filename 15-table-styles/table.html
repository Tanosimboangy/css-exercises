Table styling:

  • The default value for table-layout property is: auto.
  • The default value for border-spacing property is: 2px.
  • The default value for border-collapse property is: separate.
  • The default value for caption-side property is: top.
  • The default value for empty-cells property is: show.

Simulating HTML tables:

  • The <table> HTML element does the value corresponds to table in css.
  • The table-body or <tr> HTML element does the value corresponds to table-row-group in css.
  • The <thead> or <th> HTML element does the value corresponds to table-header-group in css.
  • The <tfoot> HTML element does the value corresponds to table-footer-group in css.
  • The <tr> HTML element does the value corresponds to table-row in css.
  • The <th> HTML element, which contains header information and <td> HTML element, which contains data do the value corresponds to table-cell in css.
  • The <colgroup> HTML element does the value corresponds to table-column-group in css.
  • The <col> HTML element does the value corresponds to table-column in css.
  • The <caption> HTML element does the value corresponds to table-caption in css.

Answearing the fallowing questions:

  • Border-spacing requires border-collapse: separate before it can work effectively.
  • The downside of using the various table values for display on a set of elements, there by similating a table-layout on non-table elements are: we migh not have the right element to be appropriate with the layout.
  • The difference between table-layout: fixed and table-layout: auto is:
    • For table-layout: fixed the table's layout overrides and uses the width of columns, border, and cell spacing values.
    • Whereas by using table-layout: auto the browser will define how the table's rows, cells, and the columns should be laid out.
  • We can use border-spacingand other table-like properties on non-table elements.
  • Empty-cells requires border-collapse before it can work effectively.
