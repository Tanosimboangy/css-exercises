overflow

  • Overflow:
    • The default value for the property overflow is visible.
    • We would use this property if we want to specify what should happen if content overflows an element's box.
    • This property is useful, because it helps the user to make the content to fit the space it belongs to.
    • Other propertie needs to be set before this propertie can work effectively is display: block.
  • Clip:
    • The default value for the clip property is auto.
    • We would use this property to control the visibility of the element which overflowed of the containt box.
    • This property is useful because it allows the user to cover the content of an element.
    • Other propertie needs to be set before this propertie can work effectively is position:absolute or position:fixed.
  • Clip-path:
    • The default value for the clip-path property is none.
    • We would use this property if we want to specify a specific region of an element to display rather than showing the complete area.

More general questions:

  • The browser's default way to handle contents that are larger than its container is that tha browser will grow the box as large as it needs to fit the content. This only happens if the width or the height is not set. But if you do specify the width or the height of the box, the css overflow will help the user to deal with that.
  • Ellipsis:
    • The ellipsis is a character that represents the clipped text and displayed inside the content area, decreasing the amount of text displayed.
    • That is different from ellipse, because ellipse is a circle.
  • Text-overflow is different from the other properties because it only happens when the container's overflow property has the value hidden, scroll or auto. Apart from that the text overflow only occurs on block or inline-block level elements, because the element needs to have a width so that it can be overflowed.
