Positioning

  • The default value for the property Position is static\.

Answer these questions too:

  • Some of the other propreties the user needs to supply when using any non-default position value are:
    • Width
    • Top
    • Right
    • Bottom
    • Left
  • The inferred display value of an element that has been positioned is block.
  • One of the non-default values for position still maintains a box in the flow of the document. It is the relative value.
  • The special or significant about Fixed and Sticky positioning is that:
    • They keep the element to stay on the same location even though the document is scrowlled up or down.
    • They can stop the viewport from changing when the window is scrolled, and fixed or sticky positioned element will be on the place where it is when the page is scrolled.
    • However, they still allow us to scrowl the viewport without any difficulties.
  • When positioning an element absolutely, to lock that element's coordinates to its parent element we need to set a specific coordinates for its parents before setting up the coordinates for its children.
  • Absolute positioned layouts are useful for:
    • Placing proprely any page element exactly where you want it to be.
    • By using the positioning attributes top, left, bottom, and right the user can set the location perfectly.
    • Controling the layout exactly as what you want it to be displayed.
  • Sometimes if you put two absolutely positioned elements in the same physical space, they'll stack, i.e. only one will be visible on top:
    • The rules here are:
      • Two elements set absolutely on the same place, if they have the same size, the one which is set the first will occupy the place.
      • Using the property z-index helps the user to position two or more elements on the same location, but they have different sizes.
    • A certain element is on top because this is the first element that set the first.
    • We can adjust that ourselves by using the property z-index, which sets the elements to appear on the same location according to their orders.
  • CSS's Stacking Context is an element that contains a set of layers.
