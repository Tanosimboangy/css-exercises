Custom Properties

  • We have to use some characters when defining a Custom Property's name. Those are:
    • Two dashes(--) preceded the name.
    • Dashe(-) or underscore(_) if the name is more than two words.
  • Characters we can't use in a Custom Property name are:
    • Spaces.
    • Period or fullstop.
  • The possible values we can assign to a new Custom Property are:
    • Number
    • Text
    • Key words
  • It is useful to create Custom Properties using the :root or html selector when we want to avoid rewrite the same values every property. To be clear, the user can reduce the need of repetition. Furthermore, it can be applied across your HTML document. Using Custom Properties keeps you to risk for having an error.
    • It is not when we want to use the same variable custom property but has differnt values in each time.
  • After we've defined a new Custom Property, we can recall or use that Custom Property. For that to happen, after we've set the selector, you have to specify your custom property name inside the var() function as a value of the property.
    • We can use var() function.
  • There are several ways to override or change a Custom Property. These are some of them:
    • Specify the selector and set the new custom property with the new selector and the new value for it. But it only works on div.
    • We can change it directly.
    • Using @media.
    • Set the property under the last one.
    • Custom Properties are useful because they help the author to write less and clean but really powerful css.
    • Moreover, they can help to avoid the repetition of some same values.
    • One point of control.
  • The only aspect (or part) of CSS syntax that we can use Custom Properties for is declaration block.
