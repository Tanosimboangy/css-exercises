Research about transition

  • Name of the functions we can use in a transform's value are:
    1. rotate()
    2. scale()
    3. translate()
    4. skew()
    5. matrix()
    6. persrective()
  • We have to use separate functions, in order, and as values, e.g. transform: scale(0.5) rotate(45deg), because using properties like: transform-scale: 0.5;, or transform-rotate: 45deg; because the order of the properties become a problem for the browser to read the code.
  • Often, we want to move the central pin or axis of the transform away from its default position. The property we use to set that is translate property.
  • It is preferable to use transform: translate() rather than position and its associated properties because using transform simplifies the condition how we can position one element by using position property, whereas chanses are there that we might have to use margin negative in order to position on element in the center of the location when we use position. Moreover, it's a bit complicated to use position.
  • When an element is translated / transformed, what happens to the flow of the document is that the transformed or the translated element will lay on top of another element which located in the same location. Therefore, the layout method similar to transform is positionning.
  • The problems I can run in to when overriding a transform in a more specific selector is that there might be a conflict if you set the tranlate property with the a but you don't set it with the a:hover. So the element will rotate but go to the default position.
  • If we want to go 3D, we need to use another property to set the view over the scene. That property is perspective.
  • We can control what the transform uses in its calculations in terms of the box model. The property we can use to adjust this is transform-box.
