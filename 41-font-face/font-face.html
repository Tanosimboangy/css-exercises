Font-face

  • There are several types of font formats, e.g. WOFF, TTF, etc.
    • Woff(Open Font Format)
    • Woff2(Web Open Font format 2.0)
    • eot(Embedded Open type)
    • otf(Open type Font)
    • ttf(True Type Font)
    • svg
  • Woff is one of these font file formats which is a good candidate to use in all browsers.
  • The font-face syntax is also different to other @-blocks. The differences between a @font-face block and @keyframes / @media queries / @supports blocks are:
    • For @font-face block we can donload the value when it does not exist in our computer, but for the others the value has to exist in the computer. We do not need a name. We do not specify the starting point and the ending point. Font face just has one curly braces. This needs source value.

  • Most of our users would probably have the exact font we want to serve already installed on their devices. To make sure the user's browser doesn't download the fonts they might already have we can specify the local function => local(......), the font which is already installed in the user's computer.
  • Sometimes we have to assign fallback fonts for our custom webfonts we add via @font-face. Some things to keep in mind when it comes to selecting these fallback fonts are they have to have the same characteristics.
  • The browser can use one of several strategies for loading the fonts, e.g. showing nothing until the font is downloaded, or using a fallback font first. We can use display property with its value fallback or swap, or we can use blank until downloaded.
